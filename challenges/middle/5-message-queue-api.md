# Backend Challenge - Message Queue API

## Introduction

Implement a message queue system for asynchronous task processing.

This challenge is tech-agnostic. Choose your preferred programming language, framework, database, ORM, message queue, and caching solution. For middle level, you can use advanced features like caching.

### Objectives

- Set up message producers and consumers
- Handle asynchronous message processing
- Ensure message reliability

### Instructions

1. **Environment Setup**:
    - Programming language and web framework
    - Database and ORM
    - Message queue system
    - Caching solution
    - Docker (recommended for containerization)

2. **Implementation Details**:
    - Create producers to send messages to queue
    - Implement consumers to process messages
    - Handle message acknowledgments and retries

3. **Testing**:
    - Send messages and verify processing

### Possible Improvements

- Add dead letter queues
- Implement message prioritization

## Conclusion

By completing this challenge, you will learn message queue fundamentals.

Happy coding!
