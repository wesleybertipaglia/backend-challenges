# Backend Challenge - Event-Driven Architecture

## Introduction

Build an event-driven system with decoupled services communicating through events.

This challenge is tech-agnostic. Choose your preferred programming language, framework, database, ORM, message broker, and event processing tools. For senior level, you can use any technologies and patterns.

### Objectives

- Design event-driven workflows
- Implement event producers and consumers
- Handle event versioning and evolution

### Instructions

1. **Environment Setup**:
    - Programming language and web framework
    - Database and ORM
    - Message broker (Kafka, RabbitMQ, etc.)
    - Event schema registry
    - Stream processing tools

2. **Implementation Details**:
    - Define event schemas and contracts
    - Implement event publishers and subscribers
    - Set up event replay and state management

3. **Testing**:
    - Test event flows and system decoupling

### Possible Improvements

- Add event sourcing
- Implement complex event processing
- Set up event-driven sagas

## Conclusion

By completing this challenge, you will master event-driven design.

Happy coding!
