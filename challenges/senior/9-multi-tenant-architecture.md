# Backend Challenge - Multi-Tenant Architecture

## Introduction

Build a multi-tenant application that serves multiple clients with data isolation.

This challenge is tech-agnostic. Choose your preferred programming language, framework, database, ORM, and tenancy models. For senior level, you can use any technologies and patterns.

### Objectives

- Implement tenant data isolation
- Manage tenant-specific configurations
- Ensure security and performance across tenants

### Instructions

1. **Environment Setup**:
    - Programming language and web framework
    - Database and ORM
    - Multi-tenant database strategy
    - Authentication and authorization
    - Caching solution

2. **Implementation Details**:
    - Implement tenant identification and context
    - Set up database isolation (separate databases, schemas, or row-based)
    - Create tenant management APIs

3. **Testing**:
    - Test tenant isolation and performance

### Possible Improvements

- Add tenant provisioning automation
- Implement tenant-specific features
- Set up cross-tenant analytics

## Conclusion

By completing this challenge, you will understand multi-tenant application design.

Happy coding!
